# Node Collapse Drag Controls Polish - 需求规格

## 概述

补齐折叠节点的可展开计数入口，稳定拖拽排序时的画布视口和命中语义，并把选中节点的缩放控制从底部 “- / +” 快捷按钮调整为节点四角控制。

## 项目信息

- 项目名: my-mind-node-workspace
- 产品名: my-mind-node
- 架构类型: monorepo
- 需求来源: 2026-06-19 节点折叠、拖拽排序和选中节点控制反馈

## 用户故事

- 作为导图编辑用户，我想要折叠节点后仍能看到隐藏节点数量，并能点击该入口再次展开，以便不会丢失被折叠分支的存在感。
- 作为导图编辑用户，我想要拖拽节点时画布不闪烁、不自动缩回最小视图，以便我能连续整理结构而不被视口跳动打断。
- 作为导图编辑用户，我想要拖到节点上时是入子，拖到节点上方或下方空隙时是排序，以便拖拽语义和我的视觉判断一致。
- 作为导图编辑用户，我想要选中节点后的缩放控制出现在节点四角，而不是底部 “- / +” 按钮，以便节点周边操作更贴合直觉且不遮挡下方内容。

## 功能需求

1. [F-001] 折叠且存在隐藏子孙节点的节点必须展示一个可点击的数量按钮，数量表示当前折叠分支隐藏的子孙节点总数。
2. [F-002] 折叠数量按钮在节点处于折叠态时必须可见，不依赖先点击选中；点击后展开该节点并重新布局。
3. [F-003] 折叠数量按钮必须与现有展开/折叠 hover 控件协同，不得出现两个互相冲突的展开入口或点击命中互相遮挡。
4. [F-004] 拖拽过程中不得出现整张画布闪烁、节点突然回弹到最小画面、自动 `fitView` 或 viewport zoom/pan 被重置。
5. [F-005] 拖动节点与目标节点发生有效重合并释放时，语义为把当前节点移动为目标节点的子节点。
6. [F-006] 拖动节点位于目标节点上方或下方的排序区域且没有与目标节点重合时，语义为同级重新排序，而不是入子。
7. [F-007] 上方排序区域释放后必须排到目标节点之前；下方排序区域释放后必须排到目标节点之后，并同步更新 `children` 顺序。
8. [F-008] 拖拽提交仍必须禁止根节点移动、拖到自身、拖到自身后代、拖到被拖动集合内部等非法结构。
9. [F-009] 点击选中节点后，不再在节点下方展示 “-” 和 “+” 缩放按钮；节点缩放入口必须移动到节点四个角。
10. [F-010] 四角缩放控制必须复用现有 `node.resize` 能力，支持鼠标操作，并遵守 `nodeSizing.minScale`、`nodeSizing.maxScale` 和步进配置。
11. [F-011] 四角缩放控制必须使用 `nodrag` / `nopan` 或等价事件隔离，点击或拖动缩放时不得触发节点拖拽、画布平移或结构变更。
12. [F-012] 只读模式不得展示结构编辑入口或节点缩放控制；如果展示折叠数量，只能作为只读提示，不能提交文档变更。

## 非功能需求

- 性能: 100 节点 fixture 下拖拽排序、展开折叠和节点缩放都不得触发连续全量布局或高频 `onChange`。
- 稳定性: 拖拽释放后的结构提交、自动布局和视口保持需要拆清边界，避免 document revision 变化导致 `fitView` 反复执行。
- 可用性: 入子、排序、非法目标和折叠计数都需要明确视觉反馈；排序区域必须和用户看到的上方/下方空隙一致。
- 可访问性: 折叠数量按钮和四角缩放控制需要可访问名称、可聚焦状态和键盘可触达路径。
- 兼容性: 桌面鼠标编辑体验优先；移动端复杂缩放手势不进入本规格。
- 安全: 节点标题和数量按钮文本按纯文本渲染，不引入 HTML 注入路径。

## 验收标准

- [ ] [AC-001] 折叠含有隐藏子孙节点的节点后，节点上展示隐藏数量按钮，例如 `4` 或 `+4`。
- [ ] [AC-002] 点击折叠数量按钮后，该节点展开，隐藏子孙节点重新出现在画布中，按钮消失或回到折叠入口状态。
- [ ] [AC-003] 折叠数量按钮和现有 hover 折叠按钮不会互相遮挡；鼠标从节点移动到按钮时按钮仍可点击。
- [ ] [AC-004] 拖动节点期间画布不闪烁，React Flow viewport 的 zoom 和 pan 不被自动重置。
- [ ] [AC-005] 将节点拖到目标节点上并释放后，节点成为目标节点子节点。
- [ ] [AC-006] 将节点拖到目标节点上方空隙释放后，节点排在目标节点之前；拖到下方空隙释放后，节点排在目标节点之后。
- [ ] [AC-007] 排序释放后 `onChange` 输出的 JSON 中同级 `children` 顺序和画布显示顺序一致。
- [ ] [AC-008] 非法拖放目标不会改变文档结构，也不会留下漂移或重叠的视觉状态。
- [ ] [AC-009] 选中节点后，节点下方不再出现 “-” 和 “+” 缩放按钮。
- [ ] [AC-010] 选中节点后，节点四角出现缩放控制；操作四角控制会调整节点 scale，并保持选中态。
- [ ] [AC-011] 操作四角缩放控制时不会触发节点拖拽、画布平移或入子/排序 intent。
- [ ] [AC-012] 相关 React 组件测试和 Playwright E2E 覆盖折叠计数展开、无视口重置拖拽、上/下空隙排序和四角缩放控制。
- [ ] [AC-013] `pnpm typecheck`、`pnpm test`、`pnpm e2e`、`git diff --check` 通过。

- 暂无。隐藏数量默认统计折叠分支内全部隐藏子孙节点；如后续产品希望只显示直接子节点数量，需要先更新本规格。
