# Node Drag Interaction Polish - 需求规格

## 概述

修复 `MindMapEditor` 在 playground 体验中的节点拖拽、拖放重排、添加子节点和折叠展开入口问题，让画布编辑更接近真实导图整理工作流。

## 项目信息

- 项目名: my-mind-node-workspace
- 产品名: my-mind-node
- 架构类型: monorepo
- 需求来源: 2026-06-18 playground 体验反馈，涉及 `apps/playground/src/App.tsx` 与 `packages/react`

| 日期 | 版本 | 说明 |

## 用户故事

- 作为导图编辑用户，我想要拖动节点时节点能跟随鼠标，以便我明确知道当前正在调整哪个节点。
- 作为导图编辑用户，我想要拖动释放后导图自动回到稳定树形排版，以便不会出现节点重叠、连线错位或截图中的样式混乱。
- 作为导图编辑用户，我想要把节点拖到其他节点上成为子节点，或拖到其他节点上方/下方完成排序，以便快速整理层级结构。
- 作为导图编辑用户，我想要鼠标悬停节点时直接看到加子节点和折叠/展开入口，以便不依赖键盘快捷键也能完成常用编辑。

## 功能需求

1. [F-001] 节点拖动过程中必须跟随鼠标移动，单选拖动和多选拖动都应有实时位置反馈。
2. [F-002] 拖动过程中不得每帧写入 `MindMapDocument` 或触发全量布局；拖动释放后只提交一次结构或位置变更。
3. [F-003] 单节点拖动释放后默认回到稳定树形布局；非法目标或未命中目标时不得留下重叠、断裂或任意漂移位置。
4. [F-004] 拖动到其他节点中心区域并停留 2 秒后，目标节点闪一下并锁定为“加入为子节点”的 drop intent；释放后执行移动。
5. [F-005] 拖动到其他节点上方区域时展示插入到目标节点之前的排序预览线，释放后成为目标节点同级前一项。
6. [F-006] 拖动到其他节点下方区域时展示插入到目标节点之后的排序预览线，释放后成为目标节点同级后一项。
7. [F-007] 拖放移动必须禁止根节点移动、拖到自身、拖到自身后代、拖到被拖动集合内部等非法结构。
8. [F-008] 鼠标移动到可编辑节点上时，在节点边缘展示“➕”按钮；点击后创建该节点的子节点、选中新节点并重新排版。
9. [F-009] 鼠标移动到有子节点的节点上时，在节点连线侧展示展开/折叠按钮；点击后切换 `collapsed` 并重新排版。
10. [F-010] hover 控件、drop intent、闪烁特效和排序预览必须提供可访问名称，且不遮挡标题编辑输入框的基础操作。
11. [F-011] 只读模式不得展示加子节点入口，也不得提交拖放移动；只读模式是否展示折叠入口按现有 Viewer 行为保持安全默认。
12. [F-012] playground 需要覆盖 100 节点 fixture 的拖动、重排、添加子节点、折叠展开核心路径。

## 非功能需求

- 性能: 100 节点 fixture 下拖动时保持流畅，拖动中不触发高频 `onChange` 或历史记录堆积。
- 可用性: 拖放目标必须有明确视觉反馈，包括 hover 高亮、2 秒进度或完成闪烁、排序插入线。
- 可访问性: “Add child”“Collapse node”“Expand node”等按钮需要 `aria-label`；视觉反馈不能只依赖颜色。
- 兼容性: 桌面鼠标交互优先；移动端触控拖拽保持既有基础能力，不扩大本规格范围。
- 安全: 节点标题、状态文本和错误提示按文本渲染，不引入 HTML 注入路径。

## 验收标准

- [ ] [AC-001] 在 playground 中拖动任意节点时，节点视觉位置跟随鼠标移动。
- [ ] [AC-002] 拖动结束后没有节点重叠、连线拉断或孤立漂移，画布重新呈现稳定树形排版。
- [ ] [AC-003] 将节点拖到另一个节点中心并停留 2 秒后，目标节点出现一次完成态闪烁，释放后该节点成为目标节点子节点。
- [ ] [AC-004] 将节点拖到目标节点上半区释放后，节点排在目标节点之前；拖到下半区释放后，节点排在目标节点之后。
- [ ] [AC-005] 非法拖放目标不会改变文档结构，并向 `onError` 返回可恢复错误或显示可理解反馈。
- [ ] [AC-006] 悬停可编辑节点时能看到边缘“➕”按钮，点击后新增子节点并进入可继续编辑的状态。
- [ ] [AC-007] 悬停有子节点的节点时能在连线侧看到展开/折叠按钮，点击后子树显示状态正确变化。
- [ ] [AC-008] 只读 Viewer 不暴露结构编辑入口。
- [ ] [AC-009] 相关 React 组件测试、core 命令回归测试和 Playwright E2E 通过。
- [ ] [AC-010] `pnpm typecheck`、`pnpm test`、`pnpm e2e`、`git diff --check` 通过。
