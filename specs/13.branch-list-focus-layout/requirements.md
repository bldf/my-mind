# Branch List Focus Layout - 需求规格

## 概述

当思维导图层级较深导致整体缩得过小、放大后又需要频繁拖动画布时，提供可切换的一级分支列表布局：左侧展示根节点下的一级节点列表，右侧只渲染当前一级节点对应的子树，并支持侧栏折叠、悬停展开、固定、拖拽调整宽度和可拖拽吸附的半透明切换按钮。

## 实现假设

- “节点超过 2 级”按树深度判断：根节点为第 1 级，存在 `root -> 一级节点 -> 二级节点` 时视为深度超过 2 级。
- 左侧列表始终展示文档根节点 `document.rootId` 的直接子节点；右侧画布以当前选中的一级节点作为 `viewRootId` 渲染该分支下的所有可见节点。
- 点击半透明切换按钮进入列表布局；再次点击切回普通导图布局，并恢复进入列表布局前的 `viewRootId`。
- 左侧菜单顶部的收起按钮表示收起列表侧栏；收起后右侧画布占满容器，鼠标移到左侧边缘时临时展开，点击固定按钮后列表重新固定在左侧。

## 用户故事

- 作为导图阅读用户，我希望在节点层级较深时可以按一级分支切换查看，以便不用把整张大图缩得很小也能看清楚当前分支。
- 作为导图编辑用户，我希望左侧列表能快速切换根节点下的一级节点，以便右侧只聚焦展示对应分支及其子节点。
- 作为导图编辑用户，我希望列表布局可以随时切回普通导图布局，以便需要全局结构时能恢复当前导图视图。
- 作为大屏或窄屏用户，我希望左右区域宽度可以拖动调整，以便根据分支标题长度和画布内容选择合适比例。
- 作为沉浸编辑用户，我希望左侧列表可以收起，右侧自动全屏，并且悬停左侧边缘可以临时展开列表，以便在不牺牲空间的情况下快速切换分支。
- 作为暗黑模式用户，我希望左侧列表、切换按钮、拖拽分隔条和悬停展开状态都适配暗色主题，以便整体体验一致。

## 功能需求

1. [F-001] 当文档树深度超过 2 级且根节点存在至少 1 个一级子节点时，编辑器容器左侧必须展示半透明的列表布局切换按钮；深度不超过 2 级或无一级子节点时不展示该按钮。
2. [F-002] 半透明切换按钮必须支持点击切换：普通导图布局点击后进入列表布局，列表布局再次点击后回到普通导图布局，并恢复进入前的视图根节点。
3. [F-003] 半透明切换按钮必须支持在编辑器容器内拖动；拖动释放或拖到容器边界外时，按钮位置必须被限制并吸附到容器边缘，不能丢失到不可点击区域。
4. [F-004] 列表布局必须把编辑器内容分成左右两部分：左侧为一级分支列表，右侧为当前选中分支的思维导图画布。
5. [F-005] 左侧列表内容必须来自文档根节点下的一级节点，并展示每个一级节点的标题、选中态、分支色彩提示和必要的子节点数量信息；标题过长时需要优雅截断或换行，不得撑破侧栏。
6. [F-006] 进入列表布局时，默认选中当前视图根所属的一级分支；如果无法推断，则选中根节点下第一个一级节点。
7. [F-007] 点击左侧任一一级节点后，右侧画布必须只渲染该节点及其全部可见后代，并将该一级节点作为右侧画布根节点展示。
8. [F-008] 列表布局下切换一级节点时不得修改 `MindMapDocument` 数据、不得新增 history，也不得破坏节点选择、折叠、拖拽、搜索、主题、toolbar、面包屑、MiniMap 的既有语义。
9. [F-009] 左右布局之间必须提供可拖动分隔条，用户可以调整左侧列表宽度；宽度需要有最小值、最大值和容器比例上限，避免列表或画布被压到不可用。
10. [F-010] 左侧菜单顶部必须提供收起按钮；点击后列表收起，右侧画布自动占满编辑器容器，并在布局稳定后保持当前 zoom 或重新居中到当前分支。
11. [F-011] 列表收起后，鼠标移动到容器左侧触发区域时必须临时展开列表；鼠标离开后列表重新收起，除非用户点击固定按钮。
12. [F-012] 临时展开的左侧菜单必须提供固定按钮；点击后列表固定在左侧，右侧画布恢复左右分栏布局。
13. [F-013] 列表布局、折叠态、悬停展开态、拖拽分隔条和半透明按钮必须支持浅色和暗黑主题，视觉上要与现有 toolbar、面板、节点风格一致。
14. [F-014] `MindMapViewer` 必须继承列表布局能力，但只读模式下列表切换、分支聚焦、侧栏折叠和宽度调整不得触发文档变更。
15. [F-015] 列表布局能力必须提供可选配置关闭入口，宿主应用可以禁用自动展示切换按钮，避免影响自定义嵌入场景。

## 非功能需求

- 性能: 1000 节点 fixture 下，列表生成、分支切换、侧栏拖拽和按钮拖拽保持流畅；分支切换只改变视图状态，不进行全量文档重写。
- 可用性: 切换按钮、收起按钮、固定按钮、列表项和分隔条必须有明确 `aria-label` / `title`；键盘用户至少可以通过按钮进入/退出列表布局并选择列表项。
- 兼容性: 保持现有 `MindMapEditor`、`MindMapViewer`、`onViewRootChange`、`breadcrumbs`、`toolbar`、`viewport.fitViewOnResize`、`search.hidden`、`minimap` 的向后兼容；新增配置必须是可选的。
- 响应式: 窄容器下侧栏宽度必须自动收敛，列表文本不得覆盖右侧画布；容器过窄时可以默认使用收起态或 overlay 展开态。
- 稳定性: pointer、resize、hover、document change 相关监听必须在组件卸载时清理；拖拽过程中不能留下不可恢复的全局事件。
- 安全: 不引入 HTML 注入、外部数据请求、本地存储敏感信息或浏览器权限绕过；列表标题按普通文本渲染。

## 验收标准

- [ ] [AC-001] 使用超过 2 级的导图打开 playground 时，编辑器左侧出现半透明列表布局切换按钮；使用只有 1-2 级的导图时该按钮不出现。
- [ ] [AC-002] 点击半透明按钮后进入左右分栏：左侧展示根节点下的一级节点列表，右侧画布只展示默认选中一级节点及其可见后代。
- [ ] [AC-003] 再次点击半透明按钮后退出列表布局，画布恢复普通导图布局和进入前的视图根节点。
- [ ] [AC-004] 拖动半透明按钮到编辑器边界外后释放，按钮吸附在容器边缘且仍可点击。
- [ ] [AC-005] 点击左侧不同一级节点后，右侧根节点标题变为对应列表项标题，且只渲染该分支下的节点和连线。
- [ ] [AC-006] 拖动左右分隔条可以改变左侧列表宽度；宽度不会小于可用下限，也不会挤压右侧画布到不可操作。
- [ ] [AC-007] 点击左侧顶部收起按钮后，列表隐藏，右侧画布占满编辑器；鼠标移动到左侧边缘后列表临时展开。
- [ ] [AC-008] 临时展开状态下点击固定按钮后，列表固定在左侧，鼠标移出后不会自动收起。
- [ ] [AC-009] 在暗黑主题下，列表、按钮、分隔条、选中态和 hover 态对比度清晰，不出现浅色主题残留。
- [ ] [AC-010] 列表布局下切换分支、收起列表、拖动分隔条和拖动切换按钮不会触发 `onChange`，不会新增 undo/redo history。
- [ ] [AC-011] `MindMapViewer` 只读渲染时同样可以进入列表布局和切换分支，但不展示编辑型行为或写入文档。
- [ ] [AC-012] 设置列表布局禁用配置后，即使文档深度超过 2 级也不展示半透明切换按钮。
- [ ] [AC-013] 相关 React 单元测试、Playwright E2E、`pnpm typecheck`、`pnpm test`、`pnpm e2e`、`git diff --check` 通过。
