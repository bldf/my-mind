# M2.5 Product Polish - 需求规格

## 概述

补齐 React Alpha 后的产品化体验：`OutlineEditor`、导图/大纲双向同步、搜索、检查器、备注/链接/标签/任务基础编辑、多选、框选、成组拖动、拖放重排、批量操作、空状态、错误状态和移动端基础浏览。

## 项目信息

- 项目名: my-mind-node
- 架构类型: monorepo
- 需求来源: `docs/prd-mind-map.md` M2.5

## 需求版本

| 日期 | 版本 | 说明 |
| --- | --- | --- |
| 2026-06-18 | v1 | 从 PRD v0.12 拆分 M2.5 Product Polish |

## 用户故事

- 作为知识库或项目管理产品，我想要大纲视图和导图双向同步，以便用户可在结构化文本和画布之间切换。
- 作为重度编辑用户，我想要多选、框选、成组拖动和批量操作，以便高效整理复杂导图。
- 作为产品团队，我想要默认样例、空状态、错误状态和移动端基础体验，以便示例可直接用于演示。

## 功能需求

1. [F-001] 发布 `OutlineEditor`，支持单独使用和与画布联动。
2. [F-002] 支持大纲缩进调整层级、编辑标题、折叠/展开和拖拽排序。
3. [F-003] 支持搜索标题、备注、标签，结果可定位到画布或大纲。
4. [F-004] 支持默认检查器，编辑节点备注、链接、标签、任务和基础样式。
5. [F-005] 支持单选、多选、Shift+空白拖拽框选、选择状态回调和只读选择联动。
6. [F-006] 支持多选节点成组拖动到空白区域，保持相对位置并写入 `position`。
7. [F-007] 支持多选节点拖放到目标节点，执行 `node.moveMany`，禁止自身、后代和根节点非法移动。
8. [F-008] 支持多选批量删除、改样式、折叠/展开，批量命令可撤销。
9. [F-009] 完善空状态、解析错误、导出失败、Fullscreen API 不可用等可读反馈。
10. [F-010] 支持移动端基础浏览、触控缩放、选择和基础编辑。
11. [F-011] 完成设计 QA 截图和核心路径 E2E 覆盖。

## 非功能需求

- 可用性: 常用路径顺手，不依赖宿主额外 UI 才能完成核心编辑。
- 可访问性: 颜色不作为标签和任务状态唯一表达；按钮和菜单有 ARIA 标签。
- 性能: 50 节点以内多选成组拖动无明显卡顿；拖动过程中不得频繁触发全量布局。
- 兼容性: 桌面端优先，移动端保证基础浏览和编辑，复杂拖拽可渐进增强。

## 验收标准

- [ ] [AC-001] `OutlineEditor` 与画布任一视图编辑后另一视图立即反映。
- [ ] [AC-002] 搜索结果可定位，且标题、备注、标签均可命中。
- [ ] [AC-003] 多选、框选、成组拖动、拖放到目标节点和批量操作可撤销。
- [ ] [AC-004] 默认样例无需额外 CSS 即可用于产品演示。
- [ ] [AC-005] 空状态、错误状态和移动端基础路径通过手动 QA 和 E2E。
