# M2 React Alpha - 需求规格

## 概述

发布可嵌入的 React Alpha：`MindMapEditor`、`MindMapViewer`、默认主题、右上角主题切换侧边栏、节点局部放大/缩小、工具栏、面包屑、进入节点视图、全屏、缩放控制、只读模式、快捷键和 Vite 示例。

## 项目信息

- 项目名: my-mind-node
- 架构类型: monorepo
- 需求来源: `docs/prd-mind-map.md` M2

## 需求版本

| 日期 | 版本 | 说明 |
| --- | --- | --- |
| 2026-06-18 | v1 | 从 PRD v0.12 拆分 M2 React Alpha |

## 用户故事

- 作为前端开发者，我想要复制 React 示例后 10 分钟内渲染可编辑导图。
- 作为宿主应用，我想要受控/非受控模式和 `onChange`，以便接管保存和权限。
- 作为产品团队，我想要默认主题、主题切换和节点局部缩放，以便不用额外 UI 就能演示核心体验。

## 功能需求

1. [F-001] 发布 `MindMapEditor` 和 `MindMapViewer`，支持受控/非受控、只读和 `height`。
2. [F-002] `MindMapEditor` 默认展示顶部右上角工具栏，包含主题、全屏、缩小、放大、适配视图入口。
3. [F-003] 实现右上角主题切换侧边栏，点击主题列表项立即应用主题并触发 `onThemeChange` 或 `onChange`。
4. [F-004] 实现节点局部放大/缩小，写入 `node.style.scale`，不改变画布 zoom，支持 undo/redo。
5. [F-005] 实现全屏切换，不支持 Fullscreen API 时返回结构化错误或触发 `onError`。
6. [F-006] 实现 `+`/`-` 缩放、Ctrl+滚轮按鼠标位置缩放、fit view 和空白平移。
7. [F-007] 实现进入节点视图和顶部面包屑导航，视图切换不修改文档树。
8. [F-008] 实现只读 `MindMapViewer`，隐藏编辑入口，但保留浏览、选择、搜索入口预留、缩放、全屏、进入节点和导出配置。
9. [F-009] 实现默认快捷键：Tab、Enter、Delete、F2、Space、Ctrl/Cmd+Z、Ctrl/Cmd+Shift+Z、Esc。
10. [F-010] 完成 Vite 最小示例和在线示例页面原型：左侧数据编辑区、右侧导图预览区。
11. [F-011] 完成 React 组件测试和核心浏览器 E2E 覆盖。

## 非功能需求

- 可用性: 默认组件可直接使用，不要求宿主实现工具栏。
- 可访问性: 所有按钮、菜单和节点控件提供可访问名称。
- 性能: 100 节点测试导图可流畅浏览、编辑、折叠和缩放。
- 安全: 主题、节点标题和错误状态不引入 HTML 注入路径。

## 验收标准

- [ ] [AC-001] 新项目安装后 10 分钟内可完成 `MindMapEditor` 最小集成。
- [ ] [AC-002] 右上角主题按钮可打开右侧侧边栏，点击主题项立即切换主题。
- [ ] [AC-003] 点击节点后可放大/缩小节点，变化可保存、撤销和重做，不影响画布 zoom。
- [ ] [AC-004] 全屏、缩放、Ctrl+滚轮、fit view、空白平移可在 playground 验证。
- [ ] [AC-005] 进入节点视图后面包屑可返回祖先节点，文档树结构不变。
- [ ] [AC-006] `MindMapViewer` 只读模式不暴露编辑入口。
- [ ] [AC-007] React 包 build/test/typecheck/lint 通过。
